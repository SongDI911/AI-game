# 残响编年史 · 项目逻辑与实现流程梳理

## 🎮 项目概述

这是一个基于 Pygame 的 2D 动作肉鸽游戏原型，核心特色是"残响能量"循环系统，结合了闪避/格挡/弹反机制、近战连击、自动寻的子弹、遗物系统和技能变异。

## 🏗️ 核心架构

### 1. 游戏主循环 (`src/main.py`)

```python
Game.run() 主循环:
├── 事件处理 (handle_events)
├── 时间缩放计算 (dt_scaled = dt_raw * player.time_scale * effects.time_scale)
├── 游戏状态更新
│   ├── playing: update() + draw()
│   ├── reward: draw() + RewardUI.draw()
│   └── gameover: draw() + _draw_gameover()
└── 渲染与显示
```

**关键设计:**
- **时间缩放系统**: 玩家子弹时间 + 命中停顿效果
- **状态机**: playing → reward → playing → gameover
- **离屏渲染**: 世界渲染到临时表面，再应用屏幕震动

### 2. 实体系统 (`src/game/entities.py`)

#### Player 类 - 核心玩家机制
```python
Player 状态系统:
├── 基础属性: HP, 能量, 位置, 速度
├── 战斗状态: 
│   ├── attack_cd (攻击冷却)
│   ├── attack_buffer (攻击缓冲)
│   ├── combo_step (连段计数)
│   ├── attack_action (持续攻击判定)
│   ├── dash_time (冲刺时间)
│   ├── invuln (无敌帧)
│   └── blocking (格挡状态)
├── 时间控制: time_scale, _slowmo (子弹时间)
├── 遗物系统: relics[] 列表
└── 自动射击: _auto_shoot() 寻的子弹
```

**输入处理流程:**
1. **移动输入** (WASD/方向键) → 更新速度 → 平滑朝向插值
2. **攻击输入** (鼠标左键) → 检查冷却 → 软自动瞄准 → 创建持续攻击判定
3. **格挡输入** (鼠标右键/左Shift) → 设置格挡状态和方向
4. **闪避输入** (空格) → 消耗能量 → 设置冲刺时间和无敌帧
5. **技能输入** (Q/E) → 检查能量 → 执行技能

#### Enemy 类 - AI行为系统
```python
Enemy 状态机:
├── chase (追逐) → 接近玩家 → telegraph
├── telegraph (前摇) → 显示攻击预警 → attack  
├── attack (攻击) → 瞬时伤害判定 → recover
└── recover (后摇) → 等待恢复 → chase

特殊机制:
├── 25% 概率重击 (不可格挡)
├── 精英变体 (1.6倍HP, 1.8倍韧性)
└── 燃烧DoT系统与蔓延机制
```

#### RangerEnemy - 远程敌人
```python
RangerEnemy 行为循环:
├── aim (瞄准) → 保持距离 → shoot
├── shoot (射击) → 发射投射物 → recover  
└── recover (恢复) → 重新瞄准
```

### 3. 房间管理系统 (`src/game/rooms.py`)

#### RoomManager - 房间生命周期
```python
房间流程:
1. 初始化 → Room(index) → 生成敌人
2. 游戏循环 → Room.update() → 处理战斗逻辑
3. 敌人全灭 → room_cleared = True → 触发奖励界面
4. 奖励选择 → next_room() → 生成新房间
```

#### GlobalEvents - 事件驱动架构
```python
事件分发机制:
├── DamageEvent 创建 → GlobalEvents.add_event()
├── Room.update() 消费事件 → GlobalEvents.consume()
└── 按来源分发伤害:
    ├── player 事件 → 检测敌人命中
    └── enemy 事件 → 玩家伤害处理
```

**设计优势:** 解耦攻击创建与伤害判定，支持技能快照伤害

### 4. 战斗系统 (`src/game/combat.py`)

#### Attack - 持续攻击判定
```python
攻击时序:
前摇(startup) → 生效(active) → 后摇(recovery)
                ↓
            连段窗口(can_chain)
```

**连段机制:**
- 三段连击，每段递增伤害和范围
- 攻击缓冲系统确保操作流畅性
- 软自动瞄准辅助战斗体验

### 5. 特效系统 (`src/game/effects.py`)

#### 多层次特效架构
```python
Effects 管理器:
├── HitStop (命中停顿) → 时间缩放控制
├── ScreenShake (屏幕震动) → 视觉反馈增强
├── FloatingText (飘字伤害) → 战斗信息反馈
├── ShockwaveRing (冲击波环) → 范围技能可视化
├── Particle (粒子系统) → 打击感营造
├── SlashArc (斩击拖尾) → 近战攻击轨迹
├── QuickFlash (闪光效果) → 命中瞬间反馈
└── Explosion (复合爆炸) → 死亡特效组合
```

**性能优化:**
- 特效数量上限保护 (MAX_PARTICLES, MAX_RINGS, MAX_ARCS)
- LRU缓存机制优化资源加载
- 死亡对象及时清理

### 6. 技能系统 (`src/game/skills.py`)

#### 技能变异机制
```python
LungeSkill (Q - 冲刺斩):
├── 基础: 突进60px + 范围伤害
├── 破甲: +8伤害
├── 连环: 返还50%能量
└── 幽灵: 冲刺期间无敌

ShockwaveSkill (E - 冲击波):
├── 基础: 46px范围 + 8伤害
├── 宽域: 半径增至64px
├── 连锁: 二次小范围伤害
└── 冻伤: 特殊效果预留
```

### 7. UI系统 (`src/game/ui.py`)

#### HUD - 实时信息显示
- HP/能量条可视化
- 房间进度和碎片计数
- 技能状态和遗物列表

#### RewardUI - 肉鸽成长核心
```python
奖励选择机制:
├── 2个随机遗物 + 1个技能变异
├── 遗物效果即时应用
└── 技能变异修改对应技能对象
```

## 🔄 核心游戏循环

### 帧更新流程
```python
每帧执行 (60 FPS):
1. 原始时间获取 → dt_raw
2. 特效系统更新 (真实时间) → effects.update(dt_raw)
3. 时间缩放应用 → dt_scaled = dt_raw * player.time_scale * effects.time_scale()
4. 游戏逻辑更新 (缩放时间):
   ├── RoomManager.update()
   │   ├── Room.update() → 敌人AI + 攻击判定
   │   └── 投射物更新 + 碰撞检测
   └── Player.update() → 移动 + 状态 + 自动射击
5. 渲染管线:
   ├── 世界渲染 (离屏Surface)
   ├── 屏幕震动应用
   ├── 特效覆盖层
   └── HUD绘制
```

### 战斗交互流程
```python
玩家攻击链路:
鼠标点击 → try_attack() → 创建Attack对象 → 
加入room.hitboxes → 持续判定敌人碰撞 → 
触发on_deal_damage() → 应用遗物加成 → 
伤害计算 + 特效反馈 + 能量回复

敌人攻击链路:
Enemy._perform_attack() → 创建DamageEvent → 
GlobalEvents.add_event() → Room.update()消费事件 → 
玩家on_hit_by_enemy() → 格挡/闪避/受伤判定 → 
对应特效反馈
```

## 🎯 关键设计模式

### 1. 时间缩放架构
- **多层时间控制**: `dt_scaled = dt_raw * player.time_scale * effects.time_scale()`
- **子弹时间**: 完美闪避触发 `SLOW_MO_SCALE = 0.45`
- **命中停顿**: HitStop系统提供战斗打击感

### 2. 资源回退机制
```python
assets.py 加载策略:
优先 assets/ 目录结构化资源
└── 失败时回退到 src/datas/ 序列帧
    └── 仍失败时使用几何图形占位符
```

### 3. 事件驱动解耦
- `GlobalEvents` 全局事件总线
- 攻击创建与伤害判定分离
- 支持技能快照伤害机制

### 4. 组件化设计
- **Effects系统**: 独立更新，支持时间缩放
- **Sound系统**: 可选降级，无声卡环境自动跳过
- **Fonts系统**: 跨平台中文字体智能匹配

## 🎮 游戏体验特色

### 1. 战斗深度
- **格挡系统**: 110度扇形判定，支持弹反投射物
- **完美闪避**: 120ms时间窗口，高额能量奖励
- **连段系统**: 三段式近战组合，递增伤害/范围

### 2. 肉鸽成长
- **遗物系统**: 5种遗物提供不同玩法维度
  - 易燃: 攻击附加燃烧DoT
  - 蔓延: 燃烧效果传播给附近敌人
  - Energy Surge: 能量获取+50%
  - Guard Convert: 格挡回血
  - 低血暴: 半血以下伤害+20%

### 3. 视觉反馈
- **复合特效**: 爆炸 = 闪光 + 冲击波 + 粒子 + 烟雾
- **分层渲染**: 世界 → 震动偏移 → 特效覆盖 → HUD
- **状态可视化**: 伤害数字、攻击预警、格挡方向指示

## 🛠️ 技术实现亮点

### 1. 性能优化
- **LRU缓存**: 资源加载避免重复IO
- **对象池**: 特效系统限制最大数量
- **预加载**: 游戏启动时预加载关键资源

### 2. 跨平台兼容
- **字体回退**: 智能搜索系统中文字体
- **音效降级**: 无音频环境自动静音
- **无头模式**: 支持CI/自动化测试

### 3. 开发友好
- **模块化设计**: 清晰的职责分离
- **配置集中**: `config.py` 统一管理游戏参数
- **错误容错**: 资源加载失败时的优雅降级

## 🎯 核心玩法循环

1. **探索阶段**: 玩家在房间中移动，敌人AI追踪
2. **战斗阶段**: 闪避/格挡/攻击的节奏博弈
3. **清理阶段**: 击败所有敌人获得房间奖励
4. **成长阶段**: 选择遗物或技能变异，强化build
5. **递进循环**: 进入下一房间，难度递增

这个项目展现了现代2D动作游戏的完整技术栈，从底层渲染到高层游戏逻辑，都有清晰的结构设计和实现细节。
