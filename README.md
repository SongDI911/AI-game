# 残响编年史 · Chronicles of Echoes

一个基于 Pygame 的 2D 动作肉鸽游戏原型，包含"残响能量"循环、闪避/格挡/弹反、近战连段与自动寻的子弹、遗物与技能变异、关卡制度与小游戏挑战，以及丰富的特效与音效系统。

## 🎮 游戏特色

### 核心玩法
- **残响能量循环**: 命中/格挡/完美闪避获得能量，用于释放技能
- **战斗系统**: 闪避/格挡/弹反 + 三段近战连段 + 自动寻的子弹
- **关卡制度**: 每 5 个房间为一个关卡（4 个普通房间 + 1 个 Boss 房间）
- **小游戏挑战**: 完成关卡后进入拼图小游戏，成功后进入下一关卡
- **技能变异**: 奖励选择强化技能，改变技能效果

### 战斗机制
- **格挡系统**: 110 度扇形判定，支持弹反投射物
- **完美闪避**: 120ms 时间窗口，触发子弹时间和大量能量
- **连段系统**: 三段式近战组合，递增伤害/范围
- **技能系统**: 
  - **Q - 冲刺斩**（变异：破甲/连环/幽灵/追影/裂空）
  - **E - 冲击波**（变异：宽域/连锁/冻伤/聚能/爆裂）

### 界面系统
- **主菜单**: 开始游戏、退出游戏选项
- **教程系统**: 新玩家引导，介绍游戏机制
- **暂停菜单**: 继续游戏、返回主菜单、退出
- **商店系统**: 使用时之碎片购买道具（生命药水、能量水晶等）
- **小地图**: 实时显示玩家和敌人位置
- **HUD**: HP/能量条、技能状态、遗物列表、关卡进度
- **奖励界面**: 房间清理后选择遗物或技能变异

## 📦 安装与运行

### 环境要求
- Python 3.8+
- pygame 2.5+

### 快速开始

```bash
# 克隆项目
cd never_up

# 安装依赖
pip install -r requirements.txt

# 运行游戏
python -m src.main
```

### 无头模式（用于测试/CI）

```bash
# 无窗口快速验证
HEADLESS=1 MAX_FRAMES=200 python -m src.main

# 截图验证
CAPTURE_PATH=screenshot.png CAPTURE_FRAME=100 python -m src.main
```

### Make 命令

```bash
make setup    # 安装依赖
make dev      # 运行游戏
make lint     # 代码检查
make test     # 运行测试
make build    # 打包发布
make assets   # 资源选择工具
```

## 🔄 游戏流程

```
主菜单 → 教程（首次）→ 游戏开始
    ↓
房间战斗 → 清理完成 → 奖励选择（遗物/技能变异）
    ↓
下一个房间 → ... → Boss 房间（第 5 个）
    ↓
关卡完成 → 过渡界面 → 小游戏挑战（拼图）
    ↓
小游戏完成 → 下一关卡 → ...
    ↓
玩家死亡 → 游戏结束 → 更新存档 → 返回主菜单
```

### 主循环流程

1. **事件处理**: 移动/攻击/格挡/闪避/技能输入
2. **时间缩放计算**: `dt_scaled = dt_raw * player.time_scale * effects.time_scale()`
3. **游戏逻辑更新**:
   - RoomManager 更新房间、敌人、投射物
   - Player 更新状态、自动射击
   - Effects 更新特效
4. **战斗判定**:
   - 近战持续判定与快照事件分发
   - 投射物碰撞、反弹与命中
5. **状态切换**: 房间清理 → 奖励选择 → 下一房间/关卡过渡 → 小游戏
6. **渲染**: 世界 → 屏幕震动 → 特效覆盖 → HUD

## 🛠️ 技术特点

### 性能优化
- **LRU 缓存**: 资源加载避免重复 IO
- **特效数量上限**: 粒子/冲击波/斩击弧限制
- **对象池机制**: 死亡对象及时清理

### 跨平台兼容
- **字体回退**: 智能搜索系统中文字体
- **音效降级**: 无音频环境自动静音
- **无头模式**: 支持 CI/自动化测试

### 设计模式
- **时间缩放架构**: 多层时间控制（子弹时间 + 命中停顿）
- **事件驱动解耦**: GlobalEvents 全局事件总线
- **组件化设计**: Effects/Sound/Fonts 独立模块
- **资源回退机制**: assets/ → src/datas/ 的加载策略
